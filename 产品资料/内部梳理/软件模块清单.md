## 1. 运动控制系统

管理机器人所有运动状态、步态生成、平衡保持、关节控制的完整系统。

| 属性 | DOBOT Rover X1 已知信息 | 宇树 Go2（参考） |
|------|----------------------|----------------|
| 状态机（FSM） | 有 — passive / ready / stand_down / balance_stand / walk / flying_trot 等 | 有 |
| 步态类型 | walk（行走）、flying_trot（奔跑） | walk / trot / gallop 等 |
| 姿态稳定 | 有 — balance_stand 模式，基于 IMU 数据实时校正 | 有（MPC，5ms 实时求解） |
| 动力学控制算法 | 未公开（有 balance_stand + flying_trot，底层算法未知） | MPC + WBC |
| 逆运动学 | 有 — 12 关节独立控制，底层必有 IK 求解 | 有 |
| 轮腿模式切换 | 有 — SDK 提供 `change_mode()` 接口 | Go2-W 有此功能 |
| 低层电机驱动 | 有 — DDS 接口可发布 MotorCmd 直接控制 12 关节（位置/速度/扭矩） | 有（SDK2 DDS） |

## 2. 动作引擎

预置的仿生动作序列库，可通过 SDK 调用执行。

| 属性 | DOBOT Rover X1 已知信息 | 宇树 Go2（参考） |
|------|----------------------|----------------|
| 可用动作 | dance / wave / jump / backflip / recovery | 有（类似动作库） |
| 执行方式 | `execute(*motions, loop=False)`，支持多动作串联和循环 | — |
| 速度调节 | 有 — `set_speed_ratio(ratio)`，范围 10~100 | — |
| 查询接口 | 有 — `get_motions()` 可列出所有可用动作 | — |
